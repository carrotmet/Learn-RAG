### 关键实现细节

**graph.py 链路**:
```
START → enrich ──[complete=True]──→ intent ──[needs_retrieve=True]──→ recall → fusion → hybrid_generate → END
         │                              │
         └──[complete=False]──→ follow_up ───────────────────────────────────────────────────────────→ END
                                        │
                                        └──[needs_retrieve=False]──→ generate_direct ────────────────→ END
```

**条件路由**:
- `enrich_router`: 问题完整 → intent，不完整 → follow_up
- `intent_router`: 需要检索 → recall，不需要 → generate_direct

## 建议与备注

1. **当前数据库状态**: 正式库 `rag_data.db` 有 19 个 chunks（standard 策略），召回结果有限。如需更完整测试，建议重新索引全部策略。

2. **流式接口**: `/runs/stream` 使用 LangGraph 的 `astream_events`，前端需要适配 SSE 格式接收数据。

3. **向后兼容**: `agent/graph.py` 保留了一阶段节点（index, retrieve, generate），但未连接。如需一阶段模式，可重新连接边。

4. **LLM 调用**: generate API 调用 LLM 生成答案，耗时约 5-10 秒（OpenRouter 免费 tier）。
