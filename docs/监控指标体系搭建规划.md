## 1. 体系概述

### 1.1 监控分层

```
┌─────────────────────────────────────────────────────────┐
│  Layer 3: 业务可观测（Business Observability）            │
│  - 用户满意度、任务完成率、知识库覆盖率                      │
├─────────────────────────────────────────────────────────┤
│  Layer 2: 在线监控（Online Monitoring）                   │
│  - 延迟、吞吐量、错误率、熔断状态、提示词质量                 │
├─────────────────────────────────────────────────────────┤
│  Layer 1: 离线评估（Offline Evaluation）                   │
│  - RAGAS指标、DeepEval指标、人工评估                        │
├─────────────────────────────────────────────────────────┤
│  Layer 0: 基础设施（Infrastructure）                       │
│  - 资源使用率、服务健康度、依赖可用性                         │
└─────────────────────────────────────────────────────────┘
```

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| **分层解耦** | 离线/在线/业务三层独立演进，避免强耦合 |
| **渐进建设** | 先跑通离线评估，再接入在线监控，最后融合 |
| **数据驱动** | 所有指标必须有对应数据集支撑，拒绝"感觉良好" |
| **自动反馈** | 离线指标结果自动回流到提示词优化、知识库更新 |
| **成本可控** | 免费/低成本方案优先（RAGAS、LangSmith免费额度） |

## 2. Phase 1：离线指标（RAGAS）

### 2.1 为什么先选 RAGAS

| 维度 | RAGAS 优势 |
|------|-----------|
| **成本** | 开源免费，可本地运行，无需额外API费用 |
| **覆盖** | 专为 RAG 设计，覆盖检索+生成全链路 |
| **易用** | 与 LangChain 生态无缝集成 |
| **基准** | 社区广泛采用，有公开基准可参考 |

### 2.2 RAGAS 核心指标

#### 检索质量指标（Retrieval Metrics）

| 指标名 | 含义 | 计算方式 | 目标值 | 优先级 |
|--------|------|---------|--------|--------|
| **Context Precision** | 检索结果中相关片段的比例 | 相关chunk数 / 总chunk数 | > 0.7 | P0 |
| **Context Recall** | 应被检索到的相关内容实际被检索到的比例 | 检索到的相关内容 / 所有相关内容 | > 0.6 | P0 |
| **Context Relevance** | 检索结果与问题的语义相关度 | 基于嵌入相似度 | > 0.75 | P1 |

#### 生成质量指标（Generation Metrics）

| 指标名 | 含义 | 计算方式 | 目标值 | 优先级 |
|--------|------|---------|--------|--------|
| **Faithfulness** | 回答是否忠实于检索到的上下文 | 回答中的事实可被上下文支撑的比例 | > 0.8 | P0 |
| **Answer Relevance** | 回答与问题的相关度 | 回答与问题的语义相似度 | > 0.8 | P0 |
| **Answer Correctness** | 回答的事实正确性 | 与参考答案对比（需标注数据） | > 0.75 | P1 |

#### 端到端指标（End-to-End）

| 指标名 | 含义 | 计算方式 | 目标值 | 优先级 |
|--------|------|---------|--------|--------|
| **RAGAS Score** | 综合评分（Faithfulness + AnswerRelevancy + ContextPrecision） | 加权平均 | > 0.75 | P0 |

### 2.4 代码集成示例

```python
# backend/src/evaluation/ragas_eval.py

from ragas import evaluate
from ragas.metrics import (
    faithfulness,
    answer_relevancy,
    context_precision,
    context_recall,
)
from datasets import Dataset
from langchain_openai import ChatOpenAI

class RAGASEvaluator:
    def __init__(self, api_key: str, model: str = "openai/gpt-4o-mini"):
        self.llm = ChatOpenAI(
            model=model,
            openai_api_base="https://openrouter.ai/api/v1",
            openai_api_key=api_key,
        )
        self.metrics = [
            faithfulness,
            answer_relevancy,
            context_precision,
            context_recall,
        ]
    
    def evaluate(self, dataset: Dataset) -> dict:
        """
        dataset 格式:
        {
            "question": ["什么是RAG?", ...],
            "answer": ["RAG是...", ...],
            "contexts": [["片段1", "片段2"], ...],
            "ground_truth": ["检索增强生成是...", ...]  # 可选
        }
        """
        result = evaluate(
            dataset=dataset,
            metrics=self.metrics,
            llm=self.llm,
        )
        return result.to_pandas()
    
    def evaluate_single(self, question: str, answer: str, contexts: list) -> dict:
        """单条评估，用于实时反馈"""
        dataset = Dataset.from_dict({
            "question": [question],
            "answer": [answer],
            "contexts": [contexts],
        })
        return self.evaluate(dataset)
```

### 2.5 评估触发机制

| 触发方式 | 频率 | 用途 | 实现 |
|---------|------|------|------|
| **批量评估** | 每日/每周 | 全量指标趋势 | Cron job + 历史对话日志 |
| **增量评估** | 每次新版本发布 | 回归测试 | CI pipeline |

| **人工抽检** | 每周100条 | 校准自动指标 | 标注平台 |

## 3. Phase 2：在线指标（LangSmith + 自定义）

### 3.1 LangSmith 免费额度规划

| 功能 | 免费额度 | 使用策略 |

| Prompt Playground | 无限 | 提示词迭代优化 |

### 3.2 在线监控指标体系

#### 性能指标（Performance）

| 指标名 | 类型 | 采集方式 | 告警阈值 | 优先级 |
|--------|------|---------|---------|--------|
| **Latency (P50/P95/P99)** | Gauge | Middleware计时 | P95 > 5s | P0 |
| **Throughput (QPS)** | Counter | 请求计数 | 异常下跌 > 30% | P1 |
| **Token Usage** | Counter | LLM API响应 | 单请求 > 4K | P1 |
| **Queue Depth** | Gauge | 异步队列长度 | > 100 | P2 |

#### 质量指标（Quality - 在线近似）

| 指标名 | 类型 | 采集方式 | 说明 |
|--------|------|---------|------|
| **Answer Length** | Gauge | 生成文本长度 | 异常短/长可能提示质量问题 |
| **Retrieval Empty Rate** | Rate | 检索结果为空的比例 | 高 = 知识库覆盖不足 |
| **User Feedback (+/-)** | Counter | 前端点赞/点踩 | 最直接的满意度指标 |
| **Follow-up Rate** | Rate | 用户是否追问 | 高 = 首答未解决问题 |

#### 错误指标（Error）

| 指标名 | 类型 | 采集方式 | 告警阈值 |
|--------|------|---------|---------|
| **LLM Error Rate** | Rate | API调用异常 | > 1% |
| **Embedding Error Rate** | Rate | 向量化失败 | > 0.5% |
| **Timeout Rate** | Rate | 请求超时 | > 2% |
| **Fallback Trigger Rate** | Rate | 降级到直接LLM | > 5%（知识库问题） |

### 3.4 自定义指标采集代码

```python
# backend/src/monitoring/online_metrics.py

import time
from functools import wraps
from prometheus_client import Counter, Histogram, Gauge, CollectorRegistry

# Prometheus 指标定义（或本地实现）
registry = CollectorRegistry()

# 性能指标
latency_histogram = Histogram(
    'rag_request_duration_seconds',
    'Request latency',
    ['endpoint', 'method'],
    registry=registry
)

# 质量指标
retrieval_empty_counter = Counter(
    'rag_retrieval_empty_total',
    'Empty retrieval count',
    ['query_type'],
    registry=registry
)

# 错误指标
llm_error_counter = Counter(
    'rag_llm_errors_total',
    'LLM API errors',
    ['error_type'],
    registry=registry
)

def timed(endpoint: str, method: str = "POST"):
    """装饰器：采集请求延迟"""
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            start = time.time()
            try:
                result = await func(*args, **kwargs)
                return result
            finally:
                duration = time.time() - start
                latency_histogram.labels(
                    endpoint=endpoint,
                    method=method
                ).observe(duration)
        return wrapper
    return decorator

def track_retrieval_empty(query: str):
    """记录空检索"""
    retrieval_empty_counter.labels(
        query_type=classify_query(query)  # 问题分类
    ).inc()

def track_llm_error(error: Exception):
    """记录LLM错误"""
    error_type = type(error).__name__
    llm_error_counter.labels(error_type=error_type).inc()
```

## 4. Phase 3：全链路可观测（DeepEval + 融合）

### 4.1 DeepEval 远景定位

DeepEval 提供更细粒度的评估能力，适合作为 RAGAS 的补充：

| 能力 | RAGAS | DeepEval | 建议 |
|------|-------|---------|------|
| 幻觉检测 | 基础（Faithfulness） | 多维度（Hallucination + G-Eval） | DeepEval 补充 |
| 偏见检测 | 无 | 内置 | 合规场景启用 |
| 毒性检测 | 无 | 内置 | 用户输入过滤 |
| 自定义指标 | 有限 | 灵活（G-Eval框架） | 业务指标用 DeepEval |
| 成本 | 免费 | 部分收费 | 按需启用 |

### 4.2 融合评估架构

```
┌─────────────────────────────────────────────────────┐
│  评估层（Evaluation Layer）                            │
│  ┌─────────────────┐  ┌─────────────────────────────┐  │
│  │  RAGAS（默认）   │  │  DeepEval（按需）          │  │
│  │  ├─ 日常批量评估 │  │  ├─ 幻觉深度检测            │  │
│  │  ├─ 快速反馈     │  │  ├─ 偏见/毒性扫描           │  │
│  │  └─ 成本敏感     │  │  ├─ 自定义业务指标           │  │
│  └─────────────────┘  │  └─ 合规审计                 │  │
│                       └─────────────────────────────┘  │
└─────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────┐
│  融合评分层（Fusion Layer）                            │
│  ├─ 加权融合: Score = 0.6*RAGAS + 0.4*DeepEval      │
│  ├─ 阈值决策: 任一指标低于阈值 → 触发干预              │
│  └─ 趋势分析: 多指标联合趋势判断质量退化                │
└─────────────────────────────────────────────────────┘
```

### 4.3 DeepEval 集成示例

```python
# backend/src/evaluation/deepeval_adapter.py

from deepeval import evaluate as deepeval_evaluate
from deepeval.metrics import HallucinationMetric, GEval
from deepeval.test_case import LLMTestCase

class DeepEvalAdapter:
    """DeepEval 包装器，与 RAGAS 统一接口"""
    
    def __init__(self, api_key: str, threshold: float = 0.7):
        self.threshold = threshold
        # DeepEval 默认使用 OpenAI，需配置为 OpenRouter
        # 或设置环境变量 OPENAI_API_KEY + OPENAI_API_BASE
    
    def evaluate_hallucination(self, 
                               input_text: str, 
                               actual_output: str, 
                               context: list) -> dict:
        """幻觉检测"""
        test_case = LLMTestCase(
            input=input_text,
            actual_output=actual_output,
            context=context,
        )
        metric = HallucinationMetric(threshold=self.threshold)
        metric.measure(test_case)
        return {
            "score": metric.score,
            "passed": metric.is_successful(),
            "reason": metric.reason,
        }
    
    def evaluate_custom(self,
                       input_text: str,
                       actual_output: str,
                       criteria: str) -> dict:
        """自定义指标（G-Eval框架）"""
        metric = GEval(
            name="Custom Business Metric",
            criteria=criteria,
            evaluation_params=["input", "actual_output"],
        )
        test_case = LLMTestCase(
            input=input_text,
            actual_output=actual_output,
        )
        metric.measure(test_case)
        return {
            "score": metric.score,
            "passed": metric.is_successful(),
        }
```

## 6. 功能协作机制

### 6.1 熔断机制（Circuit Breaker）

```
┌────────────────────────────────────────┐
│  熔断器状态机                           │
│                                        │
│  CLOSED ──错误率>阈值──▶ OPEN           │
│    ▲                      │              │
│    │         超时后       │ 直接返回降级回答│
│    │◀──────▶ HALF-OPEN ◀┘              │
│              探测成功                    │
│              ───────────▶ CLOSED        │
└────────────────────────────────────────┘
```

| 触发条件 | 动作 | 指标关联 |
|---------|------|---------|
| LLM Error Rate > 5% (1分钟) | 熔断 → 返回预设回答 | 在线 Error Rate |
| Latency P95 > 10s (持续30s) | 熔断 → 简化回答流程 | 在线 Latency |
| Retrieval Empty Rate > 30% | 熔断 → 直接调用LLM（无RAG） | 在线 Quality |
| Embedding API 不可用 | 熔断 → 切换到 FakeEmbeddings | 在线 Error Rate |

**代码示例：**

```python
# backend/src/monitoring/circuit_breaker.py

from enum import Enum
import time
from dataclasses import dataclass

class State(Enum):
    CLOSED = "closed"       # 正常
    OPEN = "open"           # 熔断
    HALF_OPEN = "half_open" # 探测

@dataclass
class CircuitBreaker:
    name: str
    failure_threshold: int = 5        # 连续失败次数
    recovery_timeout: float = 30.0    # 恢复探测时间（秒）
    half_open_max_calls: int = 3      # 半开状态最大探测次数
    
    def __post_init__(self):
        self.state = State.CLOSED
        self.failure_count = 0
        self.last_failure_time = None
        self.half_open_calls = 0
    
    def call(self, func, *args, **kwargs):
        if self.state == State.OPEN:
            if time.time() - self.last_failure_time > self.recovery_timeout:
                self.state = State.HALF_OPEN
                self.half_open_calls = 0
            else:
                raise CircuitBreakerOpen(f"{self.name} is OPEN")
        
        try:
            result = func(*args, **kwargs)
            self._on_success()
            return result
        except Exception as e:
            self._on_failure()
            raise
    
    def _on_success(self):
        if self.state == State.HALF_OPEN:
            self.half_open_calls += 1
            if self.half_open_calls >= self.half_open_max_calls:
                self.state = State.CLOSED
                self.failure_count = 0
        else:
            self.failure_count = 0
    
    def _on_failure(self):
        self.failure_count += 1
        self.last_failure_time = time.time()
        
        if self.state == State.HALF_OPEN:
            self.state = State.OPEN
        elif self.failure_count >= self.failure_threshold:
            self.state = State.OPEN

# 使用
llm_breaker = CircuitBreaker("llm_api", failure_threshold=3)
embedding_breaker = CircuitBreaker("embedding_api", failure_threshold=5)
```

### 6.3 提示词质量监控

| 监控维度 | 指标 | 采集方式 | 干预动作 |
|---------|------|---------|---------|
| **提示词长度** | Token数 | 计数 | 过长 → 截断/压缩上下文 |
| **提示词结构** | 模板匹配度 | 正则检查 | 不匹配 → 告警 + 回退默认模板 |
| **提示词效果** | 同一提示词不同输入的得分方差 | RAGAS评估 | 方差大 → 提示词需优化 |
| **版本对比** | A/B测试得分 | 分流评估 | 新版本得分低 → 回滚 |

**提示词版本管理：**

```python
# backend/src/prompts/registry.py

PROMPT_VERSIONS = {
    "v1.0.0": {
        "template": "基于以下文档回答问题：\n{context}\n\n问题：{question}",
        "ragas_score": 0.72,
        "usage_count": 15000,
    },
    "v1.1.0": {
        "template": "你是专业知识助手。基于提供的文档，给出清晰准确的回答。\n\n文档：\n{context}\n\n问题：{question}",
        "ragas_score": 0.78,
        "usage_count": 3000,
        "status": "canary",  # canary | stable | deprecated
    },
}

# 自动选择：canary 流量 10%，stable 流量 90%
# 若 canary 的 RAGAS score 连续7天 > stable，则提升为 stable
```

### 6.4 延迟监控与优化

| 环节 | 目标延迟 | 监控方式 | 优化手段 |
|------|---------|---------|---------|
| **检索** | < 100ms | ChromaDB查询计时 | 索引优化、缓存热点查询 |
| **Embedding** | < 200ms | API调用计时 | 批量请求、本地缓存 |
| **LLM生成** | < 3s (首token) | Streaming首token计时 | 模型降级、流式输出 |
| **总延迟** | < 5s (P95) | 端到端计时 | 异步预处理、CDN |

**延迟分解图：**

```
总延迟 5.2s
├─ 检索 0.08s  █
├─ Embedding 0.15s  ██
├─ 上下文组装 0.02s  
├─ LLM首token 2.8s  ██████████████████
├─ LLM生成 1.5s  ██████████
└─ 后处理 0.05s  

优化方向：LLM首token延迟（模型/参数优化）
```
