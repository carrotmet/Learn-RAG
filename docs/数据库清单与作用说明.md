### 1. ChromaDB 向量数据库

**路径**：`backend/chroma_db/`  
**类型**：向量数据库（ChromaDB）  
**核心文件**：`chroma.sqlite3`（2.1 MB）+ 索引目录  
**配置项**：`CHROMA_DB_PATH`（`.env` 中定义）

#### 作用
- 存储文档的 **向量嵌入**（embedding vectors）
- 提供 **语义检索**（similarity search）能力
- 与 RAG 流程的 `retrieve_node` 直接关联

#### 数据结构
| 内容 | 说明 |
|------|------|
| 文档片段 | 106 条（35 条《自指学》+ 71 条《Diagonal Arguments》） |
| 向量维度 | 2048 维（`nvidia/llama-nemotron-embed-vl-1b-v2:free`） |
| 元数据 | 每段包含来源文件名、页码等信息 |

#### 代码关联
```python
# vector_store.py
self.db = Chroma(
    persist_directory=persist_dir,
    embedding_function=embedding
)

# graph.py retrieve_node
docs = vector_store.search(question, k=4)
```

#### 注意事项
- ⚠️ **切换嵌入模型必须重建**：不同模型向量维度不同，需删除 `chroma_db` 目录重新索引
- 🔒 **只读操作**：检索时不修改，索引时写入
- 📦 **独立存储**：与业务数据（SQLite）完全隔离

### 2. 生产数据库（rag_data.db）

**路径**：`backend/data/rag_data.db`  
**类型**：SQLite  
**大小**：108 KB  
**记录**：5 条（2 对话 + 1 检索日志 + 1 LLM 调用 + 1 反馈）

#### 作用
- 存储 **RAG 系统的运行数据**：对话、检索日志、LLM 调用、用户反馈
- 支持 **在线采集**（RAG 对话时自动写入）和 **手动采集**（API 调用）
- 为 **数据导出** 提供数据源

#### 表结构
| 表名 | 记录数 | 说明 | 数据来源 |
|------|--------|------|----------|
| `conversations` | 2 | 用户对话记录 | 在线采集（graph.py 后台线程） |
| `retrieval_logs` | 1 | 检索结果日志 | 在线采集 |
| `llm_calls` | 1 | LLM 调用记录 | 在线采集 |
| `user_feedback` | 1 | 用户反馈 | 手动 API（`/api/collect/feedback`） |
| `raw_data` | 0 | 离线上传原始数据 | 离线上传（uploader.py） |
| `processed_data` | 0 | 解析后的标准数据 | 解析后生成 |

#### 代码关联
```python
# app.py / graph.py
collector = SQLiteCollector(db_path="data/rag_data.db")

# 在线采集（graph.py generate_node 中）
collector.save_conversation(...)
collector.save_retrieval_log(...)
```

#### 注意事项
- ✅ **生产环境默认数据库**：代码中所有 `SQLiteCollector` 默认指向此库
- 🔄 **实时写入**：RAG 对话时通过后台线程自动采集，不阻塞主流程
- 📊 **数据量较小**：当前只有 2 条对话记录，需要积累更多数据

### 3. 测试数据库（test_rag_data.db）

**路径**：`backend/data/test_rag_data.db`  
**类型**：SQLite  
**大小**：108 KB  
**记录**：15 条（6 对话 + 1 检索 + 1 LLM + 1 反馈 + 6 处理数据）

#### 作用
- **临时测试数据库**：运行 `demo_data.py` 或其他测试脚本时动态创建
- 包含 **更多虚拟数据** 用于验证采集和导出功能
- **与生产库隔离**：避免测试数据污染生产数据

#### 表结构
与生产数据库完全一致（6 张表），但数据更多：
| 表名 | 记录数 | 说明 |
|------|--------|------|
| `conversations` | 6 | 测试对话数据 |
| `processed_data` | 6 | 测试处理数据（生产库为 0） |
| `retrieval_logs` | 1 | 测试检索日志 |
| `llm_calls` | 1 | 测试 LLM 调用 |
| `user_feedback` | 1 | 测试反馈 |
| `raw_data` | 0 | 无原始数据 |

#### 代码关联
```python
# 测试中动态创建
test_db_path = "data/test_rag_data.db"
# 不直接引用，由测试脚本创建
```

#### 注意事项
- 🧪 **测试专用**：没有直接的业务代码引用，由测试脚本使用
- 📝 **数据更丰富**：包含 6 条处理数据（processed_data），适合测试导出功能
- ⚠️ **可安全删除**：不影响生产环境

### 4. 演示数据库（rag_data_demo.db）

**路径**：`data/rag_data_demo.db`（**项目根目录下**，非 backend 目录）  
**类型**：SQLite  
**大小**：124 KB  
**记录**：49 条（15 对话 + 8 LLM + 9 检索 + 6 反馈 + 11 原始数据）

#### 作用
- **demo_data.py 专用**：运行 `python backend/src/data_collection/demo_data.py` 生成
- **最完整的虚拟数据集**：包含所有 6 张表的数据，用于演示和教学
- **位置不同**：存储在项目根目录 `data/` 下，而非 `backend/data/`

#### 表结构
| 表名 | 记录数 | 说明 |
|------|--------|------|
| `conversations` | 15 | 完整对话数据 |
| `retrieval_logs` | 9 | 完整检索日志 |
| `llm_calls` | 8 | 完整 LLM 调用记录 |
| `user_feedback` | 6 | 完整反馈记录 |
| `raw_data` | 11 | 原始上传数据（生产/测试库为 0） |
| `processed_data` | 0 | 无处理数据 |

#### 代码关联
```python

class DemoDataGenerator:
    """生成 10-30 条不同类型的虚拟 RAG 数据"""
    
    def __init__(self, db_path: str = "data/rag_data.db"):
        ...

# 运行入口

tester = DataCollectionTester(test_db_path)
```

#### 注意事项
- 🎓 **教学演示**：数据量最大，适合演示导出、分析功能
- 📂 **位置特殊**：在 `data/rag_data_demo.db`（项目根目录），而非 `backend/data/`
- 🔄 **每次运行重新生成**：demo_data.py 会先删除旧数据库再创建新数据库
- ⚠️ **不要与生产库混淆**：代码中默认引用的是 `backend/data/rag_data.db`

### 处理阶段（processing_stage）

| 阶段 | 说明 | 来源 |
|------|------|------|
| `raw` | 原始数据，未处理 | 在线采集/离线上传 |
| `parsed` | 已导入，格式统一 | 3.2 导入完成 |
| `validated` | 已解析验证 | 3.3 解析完成 |
| `cleaned` | 已清洗 | 可选阶段 |
| `testset` | 已入选测试集 | 3.4 搭建完成 |
| `invalid` | 验证失败 | 3.3 解析失败 |

| 维度 | ChromaDB | 生产数据库 | 测试数据库 | 演示数据库 |
|------|----------|------------|------------|------------|
| **类型** | 向量数据库 | SQLite | SQLite | SQLite |
| **存储** | 向量嵌入 | 业务数据 | 测试数据 | 虚拟数据 |
| **用途** | 语义检索 | 生产采集 | 功能测试 | 教学演示 |
| **数据来源** | 文档索引 | RAG 在线运行 | 测试脚本 | demo_data.py |
| **代码引用** | vector_store.py | app.py/graph.py | 无直接引用 | demo_data.py |
| **可删除** | ❌ 必须重建 | ❌ 生产数据 | ✅ 可删除 | ✅ 可删除 |
| **位置** | backend/chroma_db/ | backend/data/ | backend/data/ | data/ |

### 使用建议

| 场景 | 使用数据库 | 操作 |
|------|-----------|------|
| **日常 RAG 对话** | ChromaDB + 生产库 | 检索 + 自动采集 |
| **开发测试** | 测试数据库 | 运行 `demo_data.py` 时使用 `test_rag_data.db` |
| **教学演示** | 演示数据库 | 运行 `demo_data.py` 生成 `rag_data_demo.db` |
| **数据导出** | 生产库或演示库 | 使用 `exporter.py` 或 API 接口 |
| **切换嵌入模型** | 删除 ChromaDB | `rm -rf backend/chroma_db` 后重建 |
