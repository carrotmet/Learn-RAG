## 三、根因分析（修正版）

### 3.1 核心问题：CLI 初始化函数 `_init_components` 参数传递不完整

```python
# cli.py 中的 _init_components 函数
def _init_components(db_path: str):
    from hybrid.document_store import DocumentStore
    from hybrid.channels.vector import VectorChannel
    from hybrid.channels.fts import FTSChannel
    from hybrid.channels.graph import GraphChannel

    store = DocumentStore(db_path=db_path, chunk_size=2000, chunk_overlap=200)
    vector = VectorChannel()              # ❌ 未传 persist_dir！
    fts = FTSChannel(db_path=db_path)     # ✅ 正确
    graph = GraphChannel()                # ❌ 未传 path！
    return store, vector, fts, graph
```

**问题**：`VectorChannel()` 和 `GraphChannel()` 没有接收 `db_path` 参数，导致它们使用各自的默认路径。

### 3.2 默认路径计算错误

**VectorChannel 默认路径**：

```python
# vector.py
base = os.path.dirname(os.path.dirname(os.path.dirname(__file__)))
# __file__ = /.../RAG教学/backend/src/hybrid/channels/vector.py
# dirname 1 = /.../RAG教学/backend/src/hybrid/channels/
# dirname 2 = /.../RAG教学/backend/src/hybrid/
# dirname 3 = /.../RAG教学/backend/src/          ← 错误！指向了 backend/src/
# persist_dir = /.../RAG教学/backend/src/chroma_db_hybrid/
```

**GraphChannel 默认路径**：

```python
# graph.py
base = os.path.dirname(os.path.dirname(os.path.dirname(__file__)))
# 同理 → /.../RAG教学/backend/src/
# path = /.../RAG教学/backend/src/data/knowledge_graph.pkl
```

**DocumentStore 和 FTSChannel 路径**（正确）：

```python
# document_store.py（位于 hybrid/ 下，少一层 channels/）
base = os.path.dirname(os.path.dirname(os.path.dirname(__file__)))
# __file__ = /.../RAG教学/backend/src/hybrid/document_store.py
# dirname 3 = /.../RAG教学/backend/          ← 正确

# cli.py 显式传入 db_path = /.../RAG教学/backend/data/rag_data.db
```

### 3.3 路径漂移映射（完整版）

| 组件 | 文件位置 | dirname×3 结果 | 实际数据路径 | 状态 |
|------|---------|---------------|-------------|------|
| DocumentStore | `hybrid/document_store.py` | `backend/` | `backend/data/rag_data.db` | ✅ |
| FTSChannel | `hybrid/channels/fts.py` | `backend/src/` | `backend/src/data/rag_data.db` | ❌ |
| VectorChannel | `hybrid/channels/vector.py` | `backend/src/` | `backend/src/chroma_db_hybrid/` | ❌ |
| GraphChannel | `hybrid/channels/graph.py` | `backend/src/` | `backend/src/data/knowledge_graph.pkl` | ❌ |

**结果**：
- SQLite 业务数据（chunks, derivatives, index_status）→ `backend/data/rag_data.db` ✅
- FTS5 数据 → 取决于调用方式（CLI 传参正确，但默认路径错误）
- ChromaDB 向量数据 → `backend/src/chroma_db_hybrid/` ❌
- Graph 图谱数据 → `backend/src/data/knowledge_graph.pkl` ❌

### 3.4 为什么用户感觉"新建了数据库"

1. **用户首次运行**（第一个文档全策略）：
   - 数据写入 `backend/data/rag_data.db`（SQLite）
   - 向量写入 `backend/src/chroma_db_hybrid/`（ChromaDB）
   - 图谱写入 `backend/src/data/knowledge_graph.pkl`
   - 用户可能未注意到 `backend/src/` 下的文件

2. **用户再次运行**（第二个文档全策略）：
   - SQLite 数据追加到 `backend/data/rag_data.db` ✅
   - 但向量数据追加到 `backend/src/chroma_db_hybrid/`（与第一个文档的向量在一起）
   - 用户注意到 `backend/src/` 下有数据文件，误以为是"新建的数据库"

3. **关键认知差异**：
   - 用户认为"数据库"= 所有数据存储（SQLite + ChromaDB + Graph）
   - 实际上代码将三种数据分散到了不同位置
   - `backend/src/data/rag_data.db`（幽灵数据库）可能是之前测试脚本创建的，与 CLI 无关

### 3.5 幽灵数据库 `backend/src/data/rag_data.db` 的来源

该数据库不是 CLI 创建的，而是**测试脚本** `test_phase1.py` 创建的：


## 七、修复建议（未执行，待用户确认）

### 方案 A：修正 CLI 初始化函数（最直接）

修改 `cli.py` 中的 `_init_components`，将 `db_path` 传递给所有通道：

```python
def _init_components(db_path: str):
    from hybrid.document_store import DocumentStore
    from hybrid.channels.vector import VectorChannel
    from hybrid.channels.fts import FTSChannel
    from hybrid.channels.graph import GraphChannel

    store = DocumentStore(db_path=db_path, chunk_size=2000, chunk_overlap=200)
    
    # 基于 db_path 推导其他数据路径
    data_dir = os.path.dirname(db_path)
    vector_dir = os.path.join(data_dir, "chroma_db_hybrid")
    graph_path = os.path.join(data_dir, "knowledge_graph.pkl")
    
    vector = VectorChannel(persist_dir=vector_dir)  # ✅ 传参
    fts = FTSChannel(db_path=db_path)                # ✅ 已有
    graph = GraphChannel(path=graph_path)            # ✅ 传参
    return store, vector, fts, graph
```

### 方案 B：统一默认路径常量

在 `hybrid/__init__.py` 中定义项目根目录常量，所有组件引用：

```python
# hybrid/config.py
import os

# 基于 hybrid/__init__.py 的位置计算
# __file__ = .../backend/src/hybrid/config.py
# dirname 1 = .../backend/src/hybrid/
# dirname 2 = .../backend/src/
# dirname 3 = .../backend/
PROJECT_ROOT = os.path.dirname(os.path.dirname(os.path.dirname(os.path.abspath(__file__))))

DEFAULT_DB_PATH = os.path.join(PROJECT_ROOT, "data", "rag_data.db")
DEFAULT_CHROMA_DIR = os.path.join(PROJECT_ROOT, "chroma_db_hybrid")
DEFAULT_GRAPH_PATH = os.path.join(PROJECT_ROOT, "data", "knowledge_graph.pkl")
```

> 注意：`PROJECT_ROOT` 指向 `backend/`，不是项目根目录 `RAG教学/`。

### 方案 C：废弃默认参数，强制显式传参

所有通道类初始化时不传参则抛异常：

```python
class VectorChannel:
    def __init__(self, persist_dir: str, collection_name: str = "hybrid_docs"):
        if not persist_dir:
            raise ValueError("必须显式指定 persist_dir")
        ...
```

### 推荐方案

**组合 A + B**：
1. 立即修复 `cli.py` 的 `_init_components`（方案 A）
2. 长期统一路径常量（方案 B）
